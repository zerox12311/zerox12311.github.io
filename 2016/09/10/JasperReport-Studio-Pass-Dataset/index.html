JasperReport Studio Pass Dataset

2016.09.10 Lux Protoss

前言

JasperReport 是一套Java開發的OpenSource這種沒幫助的話就不用多說了，自己GOOGLE

這篇是給已經會用很基礎的JasperReport使用者看的，更基礎的教學之後有機會再寫

因為之前遇到一個很鳥的問題所以在這邊紀錄一下怎麼解決
問題是在主表用DataSet and Query Dialog做撈資料的動作

撈回來的Json長這樣:

總共分三層:

第一層是用來確定回傳成功或失敗的
第二層是包在第一層的value裡面的資料
第三層是value裡面資料的集合屬性，例如一個人可能會有好幾個住址，我就用List包起來

之前遇到的問題是在JasperStudio裡面主表綁定欄位需要設定一個Root Path，這就是很靠盃的點

Root Path是甚麼呢?
以上面的Json為例子，假設我把Root Path設定為value，綁定的欄位就只有在value底下的才會有值
所以在我綁定errCode的情況下，我把Root Path設定為value的話，在報表裡面會顯示null

重點來了，如果綁定是在value底下的再下一層呢??

他會很機掰的幫你把他轉成Json，也就是說假設我綁定NAME，在第一張主報表裡面顯示，他會顯示AAA
如果我綁定ADDRESS.addr他就會顯示[{“addr”:”123”},{“addr”:”456”}]

大家都說要顯示這樣的資訊應該要用List物件或者SubReoprt來顯示，但是重點是如何把主報表查詢到的結果傳到子報表呢

查了超久的stackoverflow，都寫得模糊不清，後來我參照了幾十篇的文章做交叉比對後終於給我試出來了

使用環境

使用JasperRepost Studio 6.2 version

資料來源使用WebService，呼叫.net backend api取得Json

實做

  1. 先拉一個List物件到表上
  2. 然後選Create a List using a new dataset，下一步
  3. Dataset name隨便取一個，下方選Create an empty dataset，選完後下一步
  4. 選擇Use a JRDatasource expression，下方欄位填寫

???為你主報表Root Path為基準，底下的List名稱
以我的Json為例子，我要綁定ADDRESS底下的東西，我就要把ADDRESS傳到子報表去，而我主報表的Root Path是value，所以???就寫成ADDRESS
結果長這樣

這一串的目的是把主報表查詢到的ADDRESS List物件傳到子報表去
如果你的JSON是有更多層的，可以用.接著下一層
假設我的JSON ADDRESS裡面還有一個COUNTRY的List的話，就可以用ADDRESS.COUNTRY傳出去

大部分的教學都說到這裡就可以用了，根本不知道從哪裡開始綁定，緊接著往下一步看

  1. 在子報表的DataSet裡自己手動建立一個fields，其欄位名稱為ADDRESS底下的物件名稱
    以我的JSON為例子，ADDRESS裡面有個addr，我的fields就取名addr，接著拉到子報表裡面就可以顯示addr喇!!!!

困難點

因為這是Java的工具，所以支援Java Bean直接傳，大部分stackoverflow問的問題都是用java上遇到的，而我是.net根本不一樣，再不然就是只給到上述第4點就跟你說可以用了

而且第一次使用不知道要以Root Path為基準的話綁定錯誤並不會顯示Null，而是甚麼都沒有，就像是報表上沒有那個物件似的，根本不知道是你路徑寫錯還是BUG
